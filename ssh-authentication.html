Checking for existing SSH keys

List the files in .ssh directory, if they exist:


            $ ls -al ~/.ssh

By default, the filenames of the public keys are one of the following: 


            id_rsa.pub
            id_ecdsa.pub
            id_ed25519.pub

Generating a new SSH key

Creating a new SSH key, using the email as a label:


            $ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
					

            > Generating public/private rsa key pair.

Generating a new SSH key

Accepts the default file location:


> Enter a file in which to save the key (/home/you/.ssh/id_rsa): [Press enter]

Then type a secure passphrase:


            > Enter passphrase (empty for no passphrase): [Type a passphrase]
            > Enter same passphrase again: [Type passphrase again]

Add SSH key to the ssh-agent

The ssh-agent

By default, SSH will ask you for a passphrase when you generate your key and it will use that passphrase to encrypt and decrypt your private key.

This means you'll have to enter your passphrase every time you need to use your private key. To manage this, most SSH implementations will use an agent, which keeps your decrypted key in memory. This means you'll only need to unlock it once, and it will persist until you restart.

Adding SSH key to the ssh-agent

Start the ssh-agent in the background:


            $ eval "$(ssh-agent -s)"
          

            > Agent pid 59566

** On Windows, you'll have to use Git Bash to run this eval command.

Adding SSH key to the ssh-agent

Add the SSH private key to the ssh-agent:


            $ ssh-add ~/.ssh/id_rsa

(If you created your key with a different name, replace id_rsa in the command with the name of your private key file.)

Add a new SSH key to GitHub account

Copying the SSH key to clipboard (macOS)

Copies the contents of the id_rsa.pub file to clipboard: 


              $ pbcopy < ~/.ssh/id_rsa.pub

If pbcopy isn't working, you can locate the hidden .ssh folder, open the file in your favorite text editor, and copy it to your clipboard.

Copying the SSH key to clipboard (Windows)

Copies the contents of the id_rsa.pub file to clipboard: 


              $ clip < ~/.ssh/id_rsa.pub

If clip isn't working, you can locate the hidden .ssh folder, open the file in your favorite text editor, and copy it to your clipboard.

Copying the SSH key to clipboard (Linux)

Download and install xclip (If not install)


              $ sudo pacman -S xclip

Copies the contents of the id_rsa.pub file to clipboard: 


              $ xclip -sel clip < ~/.ssh/id_rsa.pub

If xclip isn't working, you can locate the hidden .ssh folder, open the file in your favorite text editor, and copy it to your clipboard.

Adding a new SSH key to GitHub account

  • Navigate to the GitHub user settings page
  • In the user settings sidebar, click SSH and GPG keys
  • Click New SSH key
  • In the Title field, add a descriptive label for the new key
  • Paste your key into the Key field
  • Click Add SSH key
  • If prompted, confirm your GitHub password

Done!
